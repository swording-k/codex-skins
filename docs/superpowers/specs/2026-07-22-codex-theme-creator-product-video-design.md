# Codex Theme Creator 产品视频设计

## 目标

制作一支可同时用于 GitHub 官网与抖音横屏发布的产品宣传片。成片采用
Ink Press 已验收模板的镜头结构、运动参数、节奏和声音句式，但完全替换为
Codex Theme Creator 的真实界面、真实主题资产和原生视觉语言。

交付规格：1920×1080、30fps、约 36.2 秒、H.264 MP4。主版本为中文，
依靠短字卡和画面表达，不使用旁白。声音使用模板内已授权的电影感 SFX，
不加入来源不明的 BGM。

## 受众与传播场景

- GitHub 官网访客：需要在前 8 秒理解这是一款 Codex Desktop 主题管理器。
- 抖音横屏观众：需要立刻看到强视觉主题，并在 36 秒内理解选择、创作、
  微调、切换、分享和恢复默认的完整价值。
- 普通非技术用户：文案避免 CDP、注入、manifest 等实现术语。

## 核心叙事

一句话主张：把喜欢的画面，变成你的 Codex 工作台。

叙事路径：看见变化 → 选择与微调 → 用想法创作 → 本地管理与分享 →
安全恢复默认。视频只展示能够由本地真实页面或已验收素材支持的功能。

## 视觉系统

保留产品自身的视觉语言，而不是照搬模板的纸墨皮肤：

- 背景：`#0b0d10` 到 `#161b21` 的深色渐变。
- 表面：`#151a20`、`#202832`。
- 正文：`#f3f6f8`；辅助文字：`#9ba7b2`。
- 主强调色：产品 Studio 的橙色 `#e05a2a`。
- 次强调色：官网的冷蓝 `#55b6ff`，仅用于状态与少量光线。
- 字体：系统无衬线字体栈；品牌字标使用紧凑粗体，不引入纸墨衬线字。
- 材质：深色玻璃、低亮金属和克制的赛车红橙高光；不使用廉价霓虹泛光。
- 主题主角：Porsche GT3 RS 主题负责第一视觉冲击，Alpine Lake 与
  Rainforest Focus 负责展示风格跨度。

## 字幕与产品口径

功能段使用底部短字幕，单条不超过两行。只宣传 macOS Apple 芯片公开 Beta
中已支持并可由真实界面证明的功能。Windows 不作为功能卖点，不出现“稳定”、
“已验证”等暗示。结尾小字注明“非官方社区项目；主题保存在本机”。

## 素材与数据流

单一事实源为当前本地 Studio 与仓库资产：

1. 启动本地 Theme Studio，使用无敏感数据的内置演示主题。
2. 采集 2x 整页截图、关键元素透明切片和布局坐标。
3. 将素材复制到独立的视频工程，不修改产品运行逻辑。
4. Remotion 只读取固定素材和固定时间线，禁止非确定性随机数。
5. 输出成片、关键帧联系表和审片报告。

用户现有未提交的产品代码与构建产物不纳入视频提交，也不覆盖。

## 声音设计

使用模板已授权 SFX：低频品牌冲击、卡片 whoosh、控件轻触、转场扫频、
结尾 riser → impact → sparkle。所有声音钉在画面动作帧上。主版不使用 BGM，
便于官网静音播放，也避免抖音版权问题；平台发布时仍可另行叠加平台曲库音乐。

## 验收标准

- 每个功能镜头必须来自真实产品页面或仓库内已确认的主题素材。
- 3D 特写中文字清晰，无低分辨率纹理被放大的模糊感。
- 一种主动画手法只使用一次；所有关键信息落定后有静止呼吸时间。
- 导入、导出、恢复默认等画面不得制造产品不存在的结果状态。
- 每镜头检查入场中、动作峰值与落定后的静帧。
- 整片渲染后用 ffmpeg 抽帧检查接缝、裁切、字幕安全区与音画同步。
- 交付前由独立审片代理按帧号给出问题证据，问题修复后再输出终版。

## 交付物

- 1920×1080 H.264 主片 MP4。
- 无声版 MP4，便于官网自动播放。
- 关键帧联系表 PNG。
- 可复现的 Remotion 工程与渲染命令。
- 简短审片报告，明确区分“已渲染”和“已发布”；本任务不自动发布到官网或抖音。
